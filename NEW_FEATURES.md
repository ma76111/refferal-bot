## 1️⃣ نظام الإحصائيات 📊

### الوصف
لوحة إحصائيات شاملة للأدمن لمراقبة أداء البوت

### الميزات
- ✅ إحصائيات المستخدمين (إجمالي، نشطون، محظورون)
- ✅ إحصائيات المهام (نشطة، مكتملة، ملغاة)
- ✅ إحصائيات الإثباتات (معلقة، مقبولة، مرفوضة)
- ✅ إحصائيات الإيداعات والسحوبات
- ✅ إحصائيات المخالفات
- ✅ أفضل 10 مستخدمين نشطين
- ✅ حساب الصافي (الإيداعات - السحوبات)

### كيفية الاستخدام
```
الأدمن → ⚙️ لوحة التحكم → 📊 الإحصائيات
```

### مثال على الإخراج
```
📊 إحصائيات البوت الشاملة

👥 المستخدمون:
├ إجمالي المستخدمين: 150
├ المستخدمون النشطون: 145
├ المستخدمون المحظورون: 5
└ إجمالي الأرصدة: 1,250.50 USDT

📋 المهام:
├ إجمالي المهام: 75
├ المهام النشطة: 20
├ المهام المكتملة: 50
├ المهام الملغاة: 5
├ المهام المدفوعة: 60
├ مهام التبادل: 15
└ إجمالي المكافآت: 3,500.00 USDT

...
```

### الملفات
- `models/Statistics.js` - نموذج الإحصائيات
- `handlers/statisticsHandler.js` - معالج الإحصائيات

## 2️⃣ نظام التقييمات ⭐

### الوصف
نظام تقييم المستخدمين بعد إكمال المهام (من 1 إلى 5 نجوم)

### الميزات
- ✅ تقييم من 1 إلى 5 نجوم
- ✅ إضافة تعليق اختياري
- ✅ حساب متوسط التقييم
- ✅ عرض توزيع التقييمات
- ✅ عرض آخر التقييمات
- ✅ إشعار المستخدم عند التقييم
- ✅ منع التقييم المكرر لنفس المهمة

### كيفية الاستخدام

#### للمقيِّم (صاحب المهمة):
```
بعد قبول الإثبات → زر "⭐ تقييم المستخدم"
→ اختيار التقييم (1-5)
→ إضافة تعليق (اختياري)
→ إرسال
```

#### للمستخدم (عرض تقييماته):
```
القائمة الرئيسية → ⭐ تقييماتي
```

### مثال على التقييم
```
⭐ تقييماتك

📊 متوسط التقييم: 4.7/5
🔢 عدد التقييمات: 23

📈 التوزيع:
⭐⭐⭐⭐⭐ (5): 18 تقييم
⭐⭐⭐⭐ (4): 4 تقييم
⭐⭐⭐ (3): 1 تقييم
⭐⭐ (2): 0 تقييم
⭐ (1): 0 تقييم

📝 آخر التقييمات:
1. ⭐ 5/5 - @user123
   📋 مهمة الإحالة
   💬 ممتاز جداً!
```

### الملفات
- `models/Rating.js` - نموذج التقييمات
- `handlers/ratingHandler.js` - معالج التقييمات
- `database.js` - جدول `ratings`

## 3️⃣ نظام الرسائل الجماعية 📢

### الوصف
إرسال رسائل جماعية من الأدمن لجميع المستخدمين أو فئات محددة

### الميزات
- ✅ إرسال لجميع المستخدمين
- ✅ إرسال للمستخدمين النشطين فقط
- ✅ إرسال للمستخدمين المحظورين فقط
- ✅ معاينة الرسالة قبل الإرسال
- ✅ إحصائيات الإرسال (نجح، فشل)
- ✅ سجل الرسائل الجماعية
- ✅ تأخير تلقائي لتجنب حدود Telegram

### كيفية الاستخدام
```
الأدمن → ⚙️ لوحة التحكم → 📢 رسالة جماعية
→ اختيار الفئة المستهدفة
→ كتابة الرسالة
→ معاينة
→ تأكيد الإرسال
```

### الفئات المستهدفة
1. **جميع المستخدمين** - كل من سجل في البوت
2. **المستخدمون النشطون** - غير المحظورين
3. **المستخدمون المحظورون** - المحظورون فقط

### مثال على الاستخدام
```
📢 معاينة الرسالة الجماعية

🎯 الفئة: جميع المستخدمين

📝 الرسالة:
🎉 تحديث جديد!
تم إضافة نظام التقييمات الجديد
جرب الآن وقيّم تجربتك!

⚠️ هل تريد إرسال هذه الرسالة؟
[✅ إرسال الآن] [❌ إلغاء]
```

### الملفات
- `models/Broadcast.js` - نموذج الرسائل الجماعية
- `handlers/broadcastHandler.js` - معالج الرسائل
- `database.js` - جدول `broadcasts`

## 4️⃣ نظام الاستئناف 📝

### الوصف
السماح للمستخدمين المحظورين بتقديم استئناف لرفع الحظر

### الميزات
- ✅ تقديم استئناف مع شرح السبب
- ✅ مراجعة الاستئنافات من الأدمن
- ✅ قبول أو رفض الاستئناف
- ✅ إشعار المستخدم بالنتيجة
- ✅ رفع الحظر تلقائياً عند القبول
- ✅ منع الاستئنافات المكررة
- ✅ سجل الاستئنافات

### كيفية الاستخدام

#### للمستخدم المحظور:
```
القائمة الرئيسية → 📝 استئناف
→ كتابة سبب طلب رفع الحظر (20 حرف على الأقل)
→ إرسال
→ انتظار المراجعة
```

#### للأدمن:
```
⚙️ لوحة التحكم → 📋 الاستئنافات
→ عرض الاستئنافات المعلقة
→ [✅ قبول] أو [❌ رفض]
```

### مثال على الاستئناف
```
📝 تقديم استئناف

⚠️ أنت محظور حالياً: حظر دائم
📋 السبب: مخالفات متكررة

💬 اشرح سبب طلب رفع الحظر:
```

### مثال على المراجعة (للأدمن)
```
🔔 استئناف جديد

🆔 رقم الاستئناف: 5
👤 المستخدم: @user123 (123456789)

📝 السبب:
أعتذر عن المخالفات السابقة
أتعهد بالالتزام بقواعد البوت
أرجو منحي فرصة أخرى

[✅ قبول] [❌ رفض] [👤 عرض المستخدم]
```

### الملفات
- `models/Appeal.js` - نموذج الاستئنافات
- `handlers/appealHandler.js` - معالج الاستئنافات
- `database.js` - جدول `appeals`

## 📊 قاعدة البيانات

### الجداول الجديدة

#### 1. جدول `ratings`
```sql
CREATE TABLE ratings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id INTEGER NOT NULL,
  rater_user_id INTEGER NOT NULL,
  rated_user_id INTEGER NOT NULL,
  rating INTEGER NOT NULL CHECK(rating >= 1 AND rating <= 5),
  comment TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(task_id, rater_user_id)
)
```

#### 2. جدول `broadcasts`
```sql
CREATE TABLE broadcasts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  admin_id INTEGER NOT NULL,
  message TEXT NOT NULL,
  target_type TEXT DEFAULT 'all',
  target_ids TEXT,
  status TEXT DEFAULT 'pending',
  sent_count INTEGER DEFAULT 0,
  failed_count INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME
)
```

#### 3. جدول `appeals` (موجود مسبقاً، تم تحسينه)
```sql
CREATE TABLE appeals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  ban_id INTEGER NOT NULL,
  reason TEXT NOT NULL,
  status TEXT DEFAULT 'pending',
  reviewed_by INTEGER,
  review_note TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  reviewed_at DATETIME
)
```
