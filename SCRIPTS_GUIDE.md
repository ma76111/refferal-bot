# 📚 دليل السكريبتات

هذا الملف يحتوي على شرح لجميع السكريبتات المساعدة في البوت.

---

## 🔄 إضافة نقاط التبادل (add_exchange_points.js)

### الوصف
سكريبت لإضافة 6 نقاط تبادل لجميع المستخدمين الحاليين في البوت.

### الاستخدام
```bash
node add_exchange_points.js
```

### ماذا يفعل؟
- يضيف 6 نقاط تبادل لجميع المستخدمين النشطين (غير المحظورين)
- يعرض إحصائيات النقاط بعد الإضافة
- يسجل جميع العمليات في الـ logs

### متى تستخدمه؟
- عند إطلاق ميزة جديدة وتريد منح المستخدمين نقاط مجانية
- كمكافأة للمستخدمين الحاليين
- لتشجيع استخدام نظام التبادل

### ملاحظات
- ⚠️ السكريبت يعمل على قاعدة البيانات مباشرة
- ✅ يتم إضافة النقاط فقط للمستخدمين النشطين (ban_status = 'none')
- 📊 يعرض إحصائيات شاملة بعد الإضافة

---

## 💾 النسخ الاحتياطي (backup_database.js)

### الوصف
سكريبت لعمل نسخة احتياطية كاملة من قاعدة البيانات.

### الاستخدام
```bash
node backup_database.js
```

### ماذا يفعل؟
- ينشئ نسخة احتياطية من ملف `bot.db`
- يحفظ النسخة في مجلد `backups/` مع التاريخ والوقت
- يحذف النسخ القديمة تلقائياً (يحتفظ بآخر 10 نسخ)
- يعرض قائمة بجميع النسخ الاحتياطية المتوفرة

### متى تستخدمه؟
- قبل تشغيل أي سكريبت يعدل البيانات
- قبل تحديث البوت
- بشكل دوري (يومي/أسبوعي) للحفاظ على البيانات
- قبل إجراء أي تغييرات كبيرة

### مثال على اسم النسخة
```
bot_backup_2024-05-09_14-30-45.db
```

### ملاحظات
- ✅ يتحقق من سلامة النسخة (مقارنة الحجم)
- 🗑️ يحذف النسخ القديمة تلقائياً (يحتفظ بآخر 10)
- 📁 جميع النسخ محفوظة في مجلد `backups/`
- 🔒 يمكنك نقل النسخ إلى مكان آمن خارج السيرفر

---

## 🔄 استعادة النسخة الاحتياطية

### الاستخدام اليدوي
```bash
# 1. إيقاف البوت أولاً
pm2 stop bot

# 2. نسخ النسخة الاحتياطية
cp backups/bot_backup_2024-05-09_14-30-45.db bot.db

# 3. تشغيل البوت
pm2 start bot
```

### ملاحظات مهمة
- ⚠️ يجب إيقاف البوت قبل استعادة النسخة
- ✅ تأكد من اختيار النسخة الصحيحة
- 💾 احتفظ بنسخة من الملف الحالي قبل الاستعادة

---

## 📋 نصائح عامة

### 1. النسخ الاحتياطي الدوري
```bash
# إضافة cron job للنسخ الاحتياطي اليومي (الساعة 3 صباحاً)
0 3 * * * cd /path/to/bot && node backup_database.js
```

### 2. قبل تشغيل أي سكريبت
```bash
# 1. عمل نسخة احتياطية
node backup_database.js

# 2. تشغيل السكريبت المطلوب
node add_exchange_points.js
```

### 3. التحقق من النتائج
- راجع الـ logs بعد تشغيل أي سكريبت
- تحقق من الإحصائيات المعروضة
- اختبر البوت بعد التعديلات

---

## 🆕 المستخدمون الجدد

### نقاط التبادل الافتراضية
- كل مستخدم جديد يحصل على **6 نقاط تبادل** تلقائياً عند التسجيل
- هذا يسمح له بإنشاء مهام تبادل مباشرة
- يتم إضافة النقاط في دالة `User.create()` في `models/User.js`

---

## 🔧 تعديل نقاط التبادل (للأدمن)

### كيفية الاستخدام
1. اضغط "🔍 البحث عن مستخدم" من لوحة التحكم
2. أدخل ID المستخدم
3. اضغط "🔄 تعديل نقاط التبادل"
4. أدخل عدد النقاط الجديد

### ملاحظات
- يمكن للأدمن تعديل نقاط أي مستخدم
- النقاط يمكن أن تكون أي رقم (موجب أو صفر)
- التعديل فوري ويظهر مباشرة للمستخدم

---

## ⚠️ تحذيرات مهمة

1. **قبل تشغيل أي سكريبت:**
   - عمل نسخة احتياطية
   - التأكد من فهم ما يفعله السكريبت
   - اختبار السكريبت على بيئة تجريبية إن أمكن

2. **النسخ الاحتياطية:**
   - احتفظ بنسخ احتياطية خارج السيرفر
   - راجع النسخ بشكل دوري
   - احذف النسخ القديمة جداً يدوياً

3. **السكريبتات:**
   - لا تشغل سكريبتات متعددة في نفس الوقت
   - انتظر انتهاء السكريبت قبل تشغيل آخر
   - راجع الـ logs بعد كل تشغيل

---

## 📞 الدعم

إذا واجهت أي مشكلة:
1. راجع الـ logs في `logs/`
2. تحقق من النسخ الاحتياطية
3. تواصل مع الدعم الفني
