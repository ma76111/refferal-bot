# 🎯 دليل سيناريوهات المستخدم الشامل

## 📋 جميع المسارات والاحتمالات الممكنة

---

## 1️⃣ إضافة مهمة جديدة

### المسار الأساسي
```
المستخدم → ➕ إضافة مهمة
  ↓
اختيار نوع المهمة:
  → 💰 مهمة مدفوعة
  → 🔄 تبادل إحالات
```

### 1.1 مهمة مدفوعة 💰

#### المسار الناجح
```
1. اختيار "مهمة مدفوعة"`
2. إدخال اسم البوت (نص)
3. إدخال رابط الإحالة (URL)
4. إدخال عدد الأشخاص المطلوبين (رقم 1-100)
5. إدخال المكافأة لكل شخص (رقم ≥ الحد الأدنى)
6. اختيار نوع الإثبات:
   - نص فقط
   - صور فقط
   - نص + صور
7. تأكيد إنشاء المهمة
8. ✅ تم إنشاء المهمة بنجاح
```

#### الاحتمالات والأخطاء

**أ. عند إدخال اسم البوت:**
- ✅ نص عادي → يقبل
- ❌ رقم فقط → يقبل (لكن غير منطقي)
- ❌ رموز خاصة → يقبل
- ⚠️ نص طويل جداً → يقبل (لكن قد يسبب مشاكل في العرض)

**ب. عند إدخال رابط الإحالة:**
- ✅ رابط صحيح (https://...) → يقبل
- ❌ رابط بدون https → يرفض
- ❌ نص عادي → يرفض
- ❌ رابط غير صالح → يرفض

**ج. عند إدخال عدد الأشخاص:**
- ✅ رقم من 1-100 → يقبل
- ❌ رقم < 1 → يرفض
- ❌ رقم > 100 → يرفض
- ❌ نص → يرفض
- ❌ رقم عشري (1.5) → يرفض

**د. عند إدخال المكافأة:**
- ✅ رقم ≥ الحد الأدنى → يقبل
- ❌ رقم < الحد الأدنى → يرفض
- ❌ نص → يرفض
- ✅ رقم عشري (0.5) → يقبل

**هـ. عند التأكيد:**
- ✅ الضغط على "تأكيد" → ينشئ المهمة
- ❌ الضغط على "إلغاء" → يلغي العملية
- ⚠️ الرصيد غير كافٍ → يرفض ويطلب إيداع

### 1.2 تبادل إحالات 🔄

#### المسار الناجح
```
1. اختيار "تبادل إحالات"
2. إدخال اسم البوت
3. إدخال رابط الإحالة
4. إدخال عدد الأشخاص المطلوبين
5. اختيار نوع الإثبات
6. تأكيد إنشاء المهمة
7. ✅ تم إنشاء المهمة بنجاح
```

#### الاحتمالات والأخطاء
- ⚠️ نقاط التبادل غير كافية → يرفض
- ⚠️ وصل للحد الأقصى من المهام → يرفض

## 2️⃣ تنفيذ مهمة

### المسار الأساسي
```
المستخدم → 📋 المهام المتاحة
  ↓
اختيار مهمة
  ↓
الضغط على "تنفيذ المهمة"
  ↓
فتح رابط الإحالة
  ↓
تنفيذ المهمة
  ↓
إرسال الإثبات
```

### 2.1 إرسال إثبات نصي

#### المسار الناجح
```
1. اختيار مهمة تطلب نص فقط
2. الضغط على "تنفيذ المهمة"
3. إدخال النص
4. الضغط على "إرسال"
5. ✅ تم إرسال الإثبات للمراجعة
```

#### الاحتمالات
- ✅ نص عادي → يقبل
- ✅ نص طويل → يقبل
- ❌ صورة بدلاً من نص → يرفض
- ⚠️ نص قصير جداً → يقبل (لكن قد يُرفض في المراجعة)

### 2.2 إرسال إثبات بالصور

#### المسار الناجح
```
1. اختيار مهمة تطلب صور فقط
2. الضغط على "تنفيذ المهمة"
3. إرسال صورة (1-3 صور)
4. الضغط على "إرسال"
5. ✅ تم إرسال الإثبات للمراجعة
```

#### الاحتمالات
- ✅ صورة واحدة → يقبل
- ✅ 2-3 صور → يقبل
- ❌ أكثر من 3 صور → يرفض الرابعة
- ❌ نص بدلاً من صورة → يرفض
- ❌ ملف غير صورة → يرفض

### 2.3 إرسال إثبات نص + صور

#### المسار الناجح
```
1. اختيار مهمة تطلب نص + صور
2. الضغط على "تنفيذ المهمة"
3. إرسال النص
4. إرسال الصور (1-3)
5. الضغط على "إرسال"
6. ✅ تم إرسال الإثبات للمراجعة
```

#### الاحتمالات
- ✅ نص + صورة → يقبل
- ✅ نص + 3 صور → يقبل
- ❌ نص فقط → يرفض
- ❌ صور فقط → يرفض
- ❌ أكثر من 3 صور → يرفض الزائد

### 2.4 بعد إرسال الإثبات

#### السيناريوهات المحتملة

**أ. قبول الإثبات ✅**
```
1. الأدمن يراجع الإثبات
2. يضغط "قبول"
3. المستخدم يحصل على المكافأة
4. يظهر زر "تقييم صاحب المهمة"
5. ✅ اكتمل
```

**ب. رفض مع فرصة ثانية ⚠️**
```
1. الأدمن يراجع الإثبات
2. يضغط "رفض مع فرصة ثانية"
3. يكتب سبب الرفض
4. المستخدم يحصل على 30 دقيقة للتحسين
5. خياران:
   → تحسين الإثبات (يعيد الإرسال)
   → رفض التحسين (يخسر المكافأة)
```

**ج. رفض نهائي ❌**
```
1. الأدمن يراجع الإثبات
2. يضغط "رفض نهائي"
3. يكتب سبب الرفض
4. يظهر للمستخدم زر "إبلاغ عن صاحب المهمة"
5. خياران:
   → إبلاغ (يرسل بلاغ للأدمن)
   → تجاهل
```

## 3️⃣ الإيداع 💳

### المسار الأساسي
```
المستخدم → 💳 إيداع USDT
  ↓
اختيار طريقة الإيداع:
  → Binance Pay ID
  → عنوان المحفظة
```

### 3.1 الإيداع عبر Binance Pay ID

#### المسار الناجح
```
1. اختيار "Binance Pay ID"
2. إدخال المبلغ (رقم)
3. إدخال Binance Pay ID
4. إدخال وقت التحويل
5. إرسال صورة الإثبات
6. ✅ تم إرسال الطلب للمراجعة
```

#### الاحتمالات
- ✅ مبلغ صحيح → يقبل
- ❌ مبلغ سالب → يرفض
- ❌ نص بدلاً من رقم → يرفض
- ✅ Binance ID صحيح → يقبل
- ⚠️ Binance ID خاطئ → يقبل (لكن قد يُرفض في المراجعة)
- ✅ وقت صحيح → يقبل
- ✅ صورة واضحة → يقبل
- ❌ بدون صورة → يرفض

### 3.2 الإيداع عبر عنوان المحفظة

#### المسار الناجح
```
1. اختيار "عنوان المحفظة"
2. إدخال المبلغ
3. إدخال TXID
4. إرسال صورة الإثبات (اختياري)
5. ✅ تم إرسال الطلب للمراجعة
```

#### الاحتمالات
- ✅ TXID صحيح → يقبل
- ⚠️ TXID خاطئ → يقبل (لكن قد يُرفض)
- ✅ بدون صورة → يقبل

### 3.3 بعد إرسال طلب الإيداع

**أ. قبول الطلب ✅**
```
1. الأدمن يراجع الطلب
2. يضغط "قبول"
3. يضاف المبلغ لرصيد المستخدم
4. ✅ اكتمل
```

**ب. رفض الطلب ❌**
```
1. الأدمن يراجع الطلب
2. يضغط "رفض"
3. يكتب سبب الرفض
4. المستخدم يحصل على إشعار
5. ❌ لم يضاف المبلغ
```

## 4️⃣ السحب 💸

### المسار الأساسي
```
المستخدم → 💸 سحب USDT
  ↓
اختيار طريقة السحب:
  → Binance Pay ID
  → عنوان المحفظة
```

### 4.1 السحب عبر Binance Pay ID

#### المسار الناجح
```
1. اختيار "Binance Pay ID"
2. إدخال المبلغ (≥ الحد الأدنى)
3. إدخال Binance Pay ID
4. إرسال صورة Binance Pay ID
5. ✅ تم إرسال الطلب للمراجعة
```

#### الاحتمالات
- ✅ مبلغ ≥ الحد الأدنى → يقبل
- ❌ مبلغ < الحد الأدنى → يرفض
- ❌ مبلغ > الرصيد → يرفض
- ✅ Binance ID صحيح → يقبل
- ✅ صورة واضحة → يقبل
- ❌ بدون صورة → يرفض

### 4.2 السحب عبر عنوان المحفظة

#### المسار الناجح
```
1. اختيار "عنوان المحفظة"
2. إدخال المبلغ
3. إدخال عنوان المحفظة
4. اختيار الشبكة (BSC)
5. ✅ تم إرسال الطلب للمراجعة
```

#### الاحتمالات
- ✅ عنوان صحيح → يقبل
- ⚠️ عنوان خاطئ → يقبل (لكن خطر!)
- ✅ شبكة BSC → يقبل

### 4.3 بعد إرسال طلب السحب

**أ. إتمام السحب ✅**
```
1. الأدمن يراجع الطلب
2. يضغط "تم الإرسال"
3. يخصم المبلغ من رصيد المستخدم
4. ✅ اكتمل
```

**ب. رفض السحب ❌**
```
1. الأدمن يراجع الطلب
2. يضغط "رفض"
3. يكتب سبب الرفض
4. يعاد المبلغ لرصيد المستخدم
5. ❌ لم يتم السحب
```

## 5️⃣ التقييم ⭐

### المسار الأساسي
```
المستخدم → زر "تقييم" (بعد قبول إثبات)
  ↓
اختيار عدد النجوم (1-5)
  ↓
إضافة تعليق (اختياري)
  ↓
✅ تم إرسال التقييم
```

### 5.1 التقييم بدون تعليق

#### المسار الناجح
```
1. الضغط على زر "تقييم"
2. اختيار عدد النجوم (1-5)
3. الضغط على "تخطي"
4. ✅ تم إرسال التقييم
```

### 5.2 التقييم مع تعليق

#### المسار الناجح
```
1. الضغط على زر "تقييم"
2. اختيار عدد النجوم (1-5)
3. كتابة تعليق
4. إرسال التعليق
5. ✅ تم إرسال التقييم
```

#### الاحتمالات
- ✅ تعليق قصير → يقبل
- ✅ تعليق طويل → يقبل
- ✅ تعليق بالعربي → يقبل
- ✅ تعليق بالإنجليزي → يقبل
- ⚠️ تعليق مسيء → يقبل (لكن قد يُحذف)

### 5.3 عرض التقييمات

```
المستخدم → ⭐ تقييماتي
  ↓
يعرض:
  - متوسط التقييم
  - عدد التقييمات
  - توزيع النجوم
  - آخر 5 تقييمات
```

## 6️⃣ الإبلاغ 🚨

### المسار الأساسي
```
المستخدم → زر "إبلاغ" (بعد رفض نهائي)
  ↓
تأكيد الإبلاغ
  ↓
✅ تم إرسال البلاغ للأدمن
```

### السيناريوهات

**أ. إبلاغ صحيح**
```
1. المستخدم يُرفض إثباته ظلماً
2. يضغط "إبلاغ"
3. يؤكد الإبلاغ
4. الأدمن يراجع البلاغ
5. قد يعاقب صاحب المهمة
```

**ب. إبلاغ كاذب**
```
1. المستخدم يُرفض إثباته بحق
2. يضغط "إبلاغ" كذباً
3. يؤكد الإبلاغ
4. الأدمن يراجع البلاغ
5. قد يعاقب المستخدم المبلغ
```

## 7️⃣ الاستئناف 📝

### المسار الأساسي (للمحظورين فقط)
```
مستخدم محظور → 📝 استئناف
  ↓
كتابة سبب الاستئناف (≥ 20 حرف)
  ↓
إرسال الاستئناف
  ↓
انتظار مراجعة الأدمن
```

### 7.1 استئناف ناجح

#### المسار
```
1. مستخدم محظور يضغط "استئناف"
2. يكتب سبب مقنع (≥ 20 حرف)
3. يرسل الاستئناف
4. الأدمن يراجع
5. الأدمن يقبل
6. ✅ يُرفع الحظر
```

### 7.2 استئناف مرفوض

#### المسار
```
1. مستخدم محظور يضغط "استئناف"
2. يكتب سبب غير مقنع
3. يرسل الاستئناف
4. الأدمن يراجع
5. الأدمن يرفض مع ملاحظة
6. ❌ يبقى محظوراً
```

#### الاحتمالات
- ✅ سبب ≥ 20 حرف → يقبل الإرسال
- ❌ سبب < 20 حرف → يرفض الإرسال
- ⚠️ استئناف معلق → لا يمكن إرسال آخر
- ⚠️ غير محظور → لا يمكن الاستئناف

## 8️⃣ إدارة المهام

### 8.1 إيقاف مهمة

```
المستخدم → 📊 مهامي
  ↓
اختيار مهمة نشطة
  ↓
الضغط على "إيقاف"
  ↓
✅ تم إيقاف المهمة
```

**الشروط:**
- ✅ لا توجد تقديمات معلقة → يمكن الإيقاف
- ❌ توجد تقديمات معلقة → لا يمكن الإيقاف

### 8.2 تفعيل مهمة

```
المستخدم → 📊 مهامي
  ↓
اختيار مهمة متوقفة
  ↓
الضغط على "تفعيل"
  ↓
✅ تم تفعيل المهمة
```

### 8.3 حذف مهمة

```
المستخدم → 📊 مهامي
  ↓
اختيار مهمة
  ↓
الضغط على "حذف"
  ↓
✅ تم حذف المهمة
```

**الشروط:**
- ✅ لا توجد تقديمات معلقة → يمكن الحذف
- ❌ توجد تقديمات معلقة → لا يمكن الحذف

### 8.4 عرض تقديمات المهمة

```
المستخدم → 📊 مهامي
  ↓
اختيار مهمة
  ↓
الضغط على "التقديمات"
  ↓
يعرض جميع التقديمات مع:
  - الحالة (معلق/مقبول/مرفوض)
  - المستخدم
  - التاريخ
  - الإثبات
```

## 🔟 السيناريوهات الخاصة

### 10.1 المستخدم المحظور

```
مستخدم محظور يحاول أي إجراء
  ↓
يظهر له رسالة الحظر
  ↓
الخيارات المتاحة:
  - 📝 استئناف فقط
  - لا شيء آخر
```

### 10.2 المستخدم المقيد

```
مستخدم مقيد من إضافة مهام
  ↓
يحاول إضافة مهمة
  ↓
يظهر له رسالة التقييد
  ↓
لا يمكنه إضافة مهام
```

### 10.3 نفاد الرصيد

```
مستخدم يحاول إنشاء مهمة مدفوعة
  ↓
الرصيد غير كافٍ
  ↓
يظهر له رسالة
  ↓
يطلب منه الإيداع أولاً
```

### 10.4 نفاد نقاط التبادل

```
مستخدم يحاول إنشاء مهمة تبادل
  ↓
نقاط التبادل غير كافية
  ↓
يظهر له رسالة
  ↓
يطلب منه تنفيذ مهام أولاً
```

### 10.5 الوصول للحد الأقصى من المهام

```
مستخدم يحاول إضافة مهمة
  ↓
وصل للحد الأقصى (5 مهام نشطة)
  ↓
يظهر له رسالة
  ↓
يطلب منه إكمال أو حذف مهمة أولاً
```

## 📊 ملخص جميع الاحتمالات

### عند إضافة مهمة:
1. ✅ نجاح → تُنشأ المهمة
2. ❌ رصيد غير كافٍ → رفض
3. ❌ نقاط تبادل غير كافية → رفض
4. ❌ وصل للحد الأقصى → رفض
5. ❌ مقيد من إضافة مهام → رفض
6. ❌ محظور → رفض
7. ❌ بيانات خاطئة → رفض

### عند تنفيذ مهمة:
1. ✅ نجاح → يُرسل الإثبات
2. ❌ نفذ المهمة سابقاً → رفض
3. ❌ مهمته الخاصة → رفض
4. ❌ مقيد من تنفيذ مهام → رفض
5. ❌ محظور → رفض
6. ❌ إثبات خاطئ → رفض

### عند الإيداع:
1. ✅ نجاح → يُرسل للمراجعة
2. ❌ بيانات خاطئة → رفض
3. ❌ محظور → رفض

### عند السحب:
1. ✅ نجاح → يُرسل للمراجعة
2. ❌ مبلغ < الحد الأدنى → رفض
3. ❌ مبلغ > الرصيد → رفض
4. ❌ بيانات خاطئة → رفض
5. ❌ محظور → رفض

### عند التقييم:
1. ✅ نجاح → يُرسل التقييم
2. ❌ قيّم سابقاً → رفض
3. ❌ ليس صاحب المهمة → رفض
4. ❌ محظور → رفض

### عند الإبلاغ:
1. ✅ نجاح → يُرسل البلاغ
2. ❌ محظور → رفض

### عند الاستئناف:
1. ✅ نجاح → يُرسل للمراجعة
2. ❌ غير محظور → رفض
3. ❌ استئناف معلق → رفض
4. ❌ نص < 20 حرف → رفض
